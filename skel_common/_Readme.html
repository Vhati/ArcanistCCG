ArcanistCCG 4.5 Readme
Introduction

ArcanistCCG is a virtual table to play any card game online. Like a real table, it won't enforce the rules, but it will let you do most of the crazy stuff CCGs may include: die rolling, tokens, deck manipulation, card rotation, notes, etc.

ArcanistFamiliar is a helper app that automates installation and downloads new expansions.

ArcanistSDK is a another app, for developers, to convert among various tables' formats.

Objects on the Table

The table itself is a scrollable surface of fixed dimensions. It can be resized indefinitely to add more empty space at the borders.

Objects added to the table initially appear at the top-left of a player's view.

An orange lock icon sometimes appears on objects that are unresponsive because they are in use.

Cards

There are four hotspots at the corners. Dragging any of them north, south, east, or west will rotate the card.

One of ArcanistCCG's idiosyncracies is the concept of local and remote flipping (via context menu when playing online). A card on the table can have its face visible to you but no one else, as if holding it in hand. When game rules would have you play the card, drag it from one arbitrary part of the table to another, then Remote Flip it to show the face to opponents. A Local Flip peeks or unpeeks at an otherwise facedown card. A regular Flip inverts its facing for everybody.

Tip: Screen real estate can be saved by using a layout of partially overlapping cards. There's no need to place them edge-to-edge if you're only paying attention to the art-half.

Decks

To draw a card, click. It will appear on the table (below by default). When clicking to draw from a facedown deck with AutoFace enabled (default), the new card will be locally flipped automatically (shift-clicking will suppress this so the drawn card will be facedown for you like for everyone else).

When AddToBottom is enabled, cards, other decks, and hand contents dragged onto the deck will be added to the bottom (shift dragging onto the deck will also have this effect).

Position Lock will prevent any accidental dragging.

Changing a deck between faceup and facedown does not by itself reverse the cards' order.

Tip: To manage a card that "contains" others, like a vehicle or out-of-game escrow, drag a deck under the card (since the card wasn't dragged, it won't be part of the deck). Then create a group to reposition them as necessary.

Tokens

There's not much to tokens. They resemble glass beads that can be dragged around.

FloatingNotes

FloatingNotes are text labels that drag like everything else. Click to edit; then press enter or click elsewhere to commit the change.

Hands

A hand is a scrollable, stretchable list for organizing large numbers of cards opponents should rarely see. Hands are not always necessity, since a locally flipped pile of loose cards may suffice.

To fill a hand, drag cards or decks into it. Dragging within the hand reorders it. Cards dragged out of a hand, onto the table are automatically made faceup to everyone (shift-drag for facedown).

Initially a hand is locally revealed only for the player who created it. In the top-right corner, there's a total card count. The title bar is right-clickable for a context menu. Enabling Hover causes the hand to maintain its apparent position when scrolling the table.

"..." switches to settings, where reticles can be dragged onto decks to enable Draw/Discard, which directly transfer cards between those decks and the hand.

Groups

Group objects are a convenience to perform a single action repeatedly and are unseen by opponents. To create a group, shift-drag across an area from an empty point on the table. Any other object whose top-left corner is within that area is affected when you later drag or right-click the group.

Typical uses include scooting several cards over, mass-rotating, or moving a card with an entourage of tokens/labels/cards piled on it. One overlooked feature is count. The Drag Filter is a whitelist to determine which objects move when dragging.

Tip: Shift-dragging in the MiniMap creates groups too.

Other Windows
JumboView

As the mouse moves over a card or faceup deck on the table, the JumboView displays the full-size image or text of the card. Right-clicking on the Jumbo tab will set which of those is preferred. Holding shift as the mouse moves will temporarily switch the image/text preference.

While an image is displayed, extra buttons appear, to scale and rotate it.

When text is displayed, you can drag-select to see words that run beyond the end of a field.

Chat

The bulk of player interaction takes place here, announcing moves. Additionally color coded notices will be logged here; urgent ones will change the chat tab's color.

Right-clicking the chat tab will save the log, clear the log, print all Counter windows' values in the log, temporarily set your alias, or undock the chat. When undocked, closing the chat window will redock it with the log intact.

MiniMap

Like in an RTS, the MiniMap has a scaled overview of the entire table. Clicking or dragging the red view rectangle will scroll the table.

Counter

A labelled number that increments/decrements. Value changes are logged in the chat.

DieRoller

Set a number of dice and sides, then roll. Results are logged in the chat when the window loses focus.

Timer

Click ">/II" to start/pause ticking. "Rst" stops and resets.

"..." switches to settings, where an initial time can be set to count down from. When a countdown finishes, the chat is notified. By default, the initial time is 00:00:00, so the timer counts up indefinitely.

Deck-Search

At the top are fields that search exactly like in the DeckBuilder. Below is the entirety or a filtered subset of the deck's contents, in either stack or alphabetical order.

Once one or more cards are selected, Remove removes them from existence, NewDeck removes them, then adds a new deck with identical cards to the table. Draw puts them on the table (honoring the deck's settings as if drawn normally).

Deck-List

A way to visually inspect all or part of a deck. Thumbnails are laid out wrapped 10 to a row, side-by-side from top to bottom and can be clicked to draw.

Unlike the search window, here you can preemptively restrict the view to only see X cards from the top or bottom.

Lobby

This window appears when acting as a NetPlay server. The server starts listening for new players immediately. The button in the upper-right can stop or resume listening.

Individual players can be selected and kicked out. If you kick yourself (or "KickAll"), the server stops, and you're back to playing solo.

Settings-Appearance

Local cosmetic settings like colors, background, and grid snapping are found here. The player alias and window layout, if saved, will be remembered after exiting.

To revert to defaults, delete appearance.xml.

Settings-Keyboard

User-customizable keyboard shortcuts are listed for various objects and windows, alongside up to two fields. The first determines the hotkey itself: click the field and press keys to set one. The second, if present, is a string argument for the shortcut (e.g., stock phrases for chat macros).

Settings-Paths

This dialog edits per-game settings, like where images and spoilers are expected, and how the JumboView should display text.

Tip: Bringing up this dialog and immediately clicking Apply reparses the spoiler and scans for added images. Faster than restarting ArcanistCCG.

Deck Construction

Top: Searchable fields. Left: Available cards. Right: Reprints of a selected card. Bottom: Deck contents.

Finding Cards

To skip through the cardlist, click it and type a letter. One can quickly browse the card list using the cursor keys (up: prev, down: next, left: pic, right: text).

The Basic search uses keywords like:

squirrel -nuts "climbs a tree" -"falls out"

They're simply words and quoted phrases with leading minus for negation. To search for a minus enclose it in quotes.

Regex searches are available by clicking the small buttons next to fields. (Advanced users only) Regular expressions only match if they match the entire string, so it is usuallly necessary to wrap a regex with ".*" wildcards. One simple application of regex is to find blank fields, which basic keywords won't do: "$" (a lone dollar sign) which represents the end of a line.

Sometimes there will be a row of buttons ("XLAGRBUW") in place of a field. Those are for Apprentice colors. The leftmost one ("X") is special. Alone, it shows multicolor cards. When others are also pressed, it only shows that combination of colors.

Once you've filled out the fields, hit enter or click Filter to search. Select a card from the left, a reprint from the right, and click add. Adding repeatedly will increment the number of card's copies in the deck below. Remove will remove ALL copies (incentive to think before you click).

Deck Contents

For each card in a deck, there will be a line like one of the following:

A deck can get by with only the Amount and Name rows. Including Set and FrontFile info will speed up loading and ensure all players see the same version of the card. On the other hand, leaving them blank lets players use their most recent reprint image.

The last two fields are rarely used. They replace the default back image with the face of another card. For double-sided cards, Set, BackName, and optionally BackFile need to be filled in manually. Alternatively it may be preferable to add the second face as a separate card, in a sideboard to swap in during play.

The DeckBuilder does nearly all the work for you. In addition, these rows are valid.

First ColumnSecond ColumnAction
//textComment
;text labelNestedDeck
>path from gamedirOverride default back image
< Revert to default back image

Press the skinny blank button to add a blank line and use them. Editing to a cell in the deck table isn't finalized until it loses focus.

Nested decks are useful for sideboards and organizing card types to monitor legality. Hovering over the total card count will reveal a tooltip listing each nested deck's count.

Importing and Exporting

Apprentice decks (text but not binary) can be imported/exported. Any nested decks will be exported as a single combined sideboard.

LackeyCCG decks can be imported.

Playing Against an Opponent

There is no central matchmaking server to find people. However, games' community forums tend to have a section for players to announce their Instant Messenger names.

The first player goes to Files-NetPlay, switches to the Server tab, and clicks Ok to listen on a port. Often this person's router will need port forwarding configured to allow incoming traffic. Then this player visits http://www.whatismyip.com/ and tells the address to his opponent over IM.

The second player goes to Files-NetPlay, fills in the IP, and clicks Ok. If this fails, try switching who serves and who connects. Or see the Hamachi section.

Hamachi VPN

Hamachi bypasses router issues by creating a fake network adapter, complete with its own IP, on a virtual LAN that will populate as players sign into Hamachi accounts. Once installed, you can use that address to connect to each other.

You can get it here.

Each game gets its own directory within ArcanisCCG/cards/. Within it, expansions each get dirs of images. The game dir also contains an image of the back of a card and text data files (see Expansion File and Spoiler File).

Game names and expansion names, by convention, lack punctuation, lack spaces, and capitalize each word.

Card image names are normal: letters, numbers, hyphens, apostrophes, commas, periods, and spaces. Drop any trailing elipses. If multiple cards within an expansion have the same name (e.g., alternate art), append a space and number before the obligatory image suffix (.jpg, .gif, etc).

Expansion File

Expan.dat is a plain text file (non-unicode, with CR-LF line endings), which can be made with any text editor. Each line consists of an expansion's abbreviation (see Spoiler File), a hyphen, and the expansion's full name (matching its folder in the Directory Tree). For example: "ABBR-FullSetName".

Abbreviations must be unique (ignoring case), so using 3 characters is a better choice than two.

Full expansion names, by convention, lack punctuation, lack spaces, and capitalize each word.

Spoiler File

The spoiler file is either in the binary Apprentice format (cardinfo.dat) or in the plain text ArcanistCCG format (MyGame_dat.txt). For now, I'll be describing the latter.

At the start is a header block of N field names, followed by a blank line. The first must be "Name". One must be "Set". The rest are up to you. Every card that follows must be describable by those N fields; if need be, you can make up conventions to squeeze information into a multi-purpose field.

FieldPotential card info
Type"Type - Subtype Subtype ; More"
Stats"1F" or "3S" or "3F/4S/8T"
Text"Bold Word - Bold Word - Bold Word\n\nNormal card text"
Flavor"Blah Blah\n\n{Legal:Gold}{Legal:8ed}{Legal:4ed}"

ArcanistCCG maps the "Name" field directly to image names, so stick to characters that are friendly to filesystems. An image is considered to belong to a card if the spoiler name, plus an optional space, plus an optional number, plus an image extension matches its file name. "Fred" will match "Fred 1.jpg", "Fred1.gif", and "Fred.png".

The "Set" field maps to expansion subdirectories that contain those images ("MyGame/FirstExpansion/Fred.jpg"). Each card later on will have a comma-separated list of abbreviations (each of those with an optional hyphen then arbitrary chars for rarity).

After the N field names and a blank line, come the cards. Each card is comprised of N lines of info followed by a blank. If a card has no info for a field, leave the line blank but it must be there. The spoiler must be a multiple of N lines (check from time to time during data entry to notice typos).

Only minimal interpretation is applied to card info: "\n" is a line break. The rest of the markup is at your discretion. For search convenience, surround symbols with chars that stand out from sentences: "{ICON}" or "{Multiplier:4}".

Tip: If you have ArcanistCCG running while you change the spoiler in a text editor, you can tell Arcanist to reload the spoiler by clicking the Settings menu, Paths, then the Apply button.

The DeckBuilder's menu offers spoiler editing in-GUI, but it's intended for one-off typo correcting. Changes aren't permanent until the spoiler is saved. To remove a card, select a name from the left, a reprint from the right, and click Delete from the menu. When attempting to add a new card, if there's already a reprint from the new card's set, or if the new card's set isn't valid, the attempt will be cancelled. For mass additions, stick to a text editor.

Apprentice Format

To make a binary dat compatible with both ArcanistCCG and Apprentice, create a text file as above, but with the following restrictions.

Exactly 8 fields: Name, Color, Rarity, Set, Type, Cost, Stats, Text, Flavor.

Color is one of the following, based on Magic: 1 White, 2 Blue, 4 Black, 8 Red, 16 Green, 64 Tan, 128 Land; or 32 Gold/Multi plus a combination of zero or more colors.

For the Cost field, Apprentice will replace the letters WUBRG with color symbols. When ArcanistCCG uses a binary Apprentice dat, it will provide a row of color buttons in the deckbuilder.

When done, use ArcanistSDK (a separate app) to convert the text into the final binary cardinfo.dat. Expan.dat needs no modification.
